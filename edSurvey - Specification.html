Назначение документа.

Документ описывает общие потребности Пользователя в системе, особенности реализаци.

При создании документа использована методика https://habrahabr.ru/post/335924/

Термины и определения.

Сайт – совокупность вычислительных и организационных ресурсов, предоставляющая систему в пользование. изменение стиля изменение стиля

Пользователь (user) — лицо, зарегистрированное в системе.

Личность (pPerson) — Пользователь расширенным набором атрибутов.

Вопрос — связка вопрос (question) и вариантов выбора (answer) пользователя на связанный вопрос.

Опросник (qQuerylist) — контейнер, связывающий несколько Вопросов в последовательность.

Задание (tTask) — Опросник с параметрами прохождения опроса и назначением конкретным пользователям и Бригадам.

Расписание (schedule) — Задание, определённое во времени прохождения.

Бригада (squadSquad) — группа Пользователей.

Опрос — серия вопросов без оценки правильности ответов.

Анкета — серия вопросов с оценкой ожидаемых или предпочтительных ответов.

Тест — серия вопросов с оценкой правильных ответов и ограничением проходного балла или % правильных ответов.

Контент — Вопросы, Опросники, Задания, учебные материалы, созданные и/или хранящиеся на Сайте.

Владелец — Пользователь Системы, который обладает полными правами на объект. Обычно создатель Контента сразу становится его Владельцем.

Приватный контент — Контент, который доступен:

  • на изменение только его Владельцу;

  • на прохождение в рамках Заданий — Пользователям в рамках назначенной Бригада в которую они включены.

Публичный контент — любая часть Контента, которая может быть использована любым Пользователем как напрямую, так и через механизм копирования с ссылкой на использованный Контент. Публичный контент не может быть использован напрямую в Приватном контенте, а только через механизм копирования с ссылкой на использованный Контент.

ND. Общее описание системы.

Проводить Опросы, Анкетирование, Тестирование.

ND1. Общие потребности.

ND1.1. Определять мнение Пользователей в заданной области.

ND1.2. Определять уровень знаний в заданной области.

ND1.3. Определять уровень отклонений от ожидаемых ответов.

ND1.4. Определять по итогам выполненных Заданий сложность материала для конкретного Пользователя и/или целевой группы.

ND2. Особые потребности.

ND2.1. Разделять любой контент на Публичный и Приватный.

ND2.2. Использовать доступный на момент создания Контент.

ND2.3. Блокировать изменение использованного Контента.

ND2.4. Отслеживать авторские права на Контент внутри системы.

ND2.5. Делать Публичный контент невидимым.

ND2.6. Деактивировать Контент.

Удаление или архивирование (в случае его использования) Контента, делая невозможным его дальнейшее использование, кроме как в уже запущенных Расписаниях.

ND3. Ограничения.

ND3.1.Иерархии корпоративного типа остутсвуют, т.е. все Пользователи равноценны.

ND3.2. Задания выдаются только по уникальной ссылке, сгенерированной сайтом для каждого Расписания.

ND3.3. Приватное Задание можно пройти только по запросу.

ND3.4. Все пользователи могут создавать как Публичный, так и Приватный контент.

Вопрос может быть одним из предопределённых типов: "Одиночный выбор" (RadioButton), Множественный выбор (CheckBox), "Путанка" (LinkedLinks).

Описание процесса

Пользователь входит в пункт меню "Вопросы" и переходит на страницу "Вопросы".

На странице "Вопросы" пользователь использует элемент интерфейса "Новый вопрос".

UC1.1.1. Создание карточки Вопроса.

Пользователь заполняет параметры Вопроса:

  • Название;

  • Полное описание;

  • выбирает значение атрибута "Публичное", если доступен выбор;

  • выбирает значение атрибута "Организация", если доступен выбор;

  • при необходимости устанавливает атрибут "Авторы".

После проверки всех параметров Вопроса, Пользователь использует элемент интерфейса:

  • "Сохранить" - чтобы сохранить запись Вопроса в БД и остаться на форме Вопроса в режиме редактирования;

  • "Очистить" - чтобы сбросить все атрибуты и параметры в начальное состояние и продолжить создание Вопроса;

  • "Завершить" - чтобы вернуться к странице "Вопросы".

UC1.1.2. Добавление нового варианта ответа.

После сохранения параметров Вопроса, Пользователь получает возможность использовать элемент интерфейса "Добавить вариант ответа". При его использовании отрабатывает один из сценариев из раздела UC2. Варианты ответа.
